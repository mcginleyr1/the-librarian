# Meta Profile: What Makes Rob Save an Article

This profile captures the CROSS-CUTTING patterns — not what topics, but what TYPE of content.

## Content types that get saved

1. **"Build X from scratch"** — file systems, databases, web frameworks, interpreters, compilers, search engines. The act of building something from primitives is inherently interesting regardless of domain.

2. **Internals and "how it actually works"** — Erlang scheduling, PostgreSQL query planning, LLM attention mechanisms, TCP state machines. Deep explanations of machinery.

3. **Practical production engineering** — Real companies solving real problems at scale. How Discord stores billions of messages. How Stripe does zero-downtime migrations. Postmortems.

4. **Cross-language insight** — When an idea from one ecosystem illuminates another. Rust patterns applied to Elixir. Haskell type theory applied to everyday code. Clojure data philosophy applied to databases.

5. **Novel algorithms and data structures** — CRDTs, rope data structures, probabilistic data structures, bloom filters, HyperLogLog. Especially when paired with implementation.

6. **Opinionated practitioners** — antirez, Martin Kleppmann, Martin Fowler, Hillel Wayne, Joe Armstrong, Rich Hickey, Jessica Kerr. People who think deeply and write clearly about craft.

7. **Security with technical depth** — Krebs-style investigations, SQLite exploitation, protocol-level vulnerabilities. Not "10 tips for better passwords."

## Content types that do NOT get saved

- Listicles ("10 things every developer should know")
- Career advice / productivity tips
- Product announcements without technical depth
- Framework comparison posts driven by benchmarks alone
- Hot takes and controversy-driven engagement bait
- Content gated behind heavy marketing
